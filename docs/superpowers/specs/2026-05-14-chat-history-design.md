# Chat History — Design Spec

**Date:** 2026-05-14  
**Status:** Approved

## Goal

Persist chat history across page refreshes using Mastra Memory backed by a local LibSQL (SQLite) file. No external infrastructure required.

## Packages

- `@mastra/libsql` — LibSQL storage provider
- `@mastra/memory` — Mastra Memory class

## Storage

`Memory` with `LibSQLStore` is added directly to `chatAgent` in `src/mastra/agents/chat-agent.ts`. A `storage.db` file is created in the project root on first run.

```typescript
import { Memory } from '@mastra/memory'
import { LibSQLStore } from '@mastra/libsql'

export const chatAgent = new Agent({
  ...
  memory: new Memory({
    storage: new LibSQLStore({ url: 'file:./storage.db' }),
  }),
})
```

`src/mastra/index.ts` does not need a system-level storage config — agent-level Memory is sufficient for chat history.

## Thread Identification

- `threadId`: UUID generated by the browser, stored in `localStorage` under the key `chat_thread_id`. Survives page refreshes. Cleared when the user starts a new chat.
- `resourceId`: Fixed string `"local-user"` — single-user local app.

## API Changes

### POST /api/chat

**Request body changes:**

Before: `{ messages: Array<{ role, content }> }`  
After: `{ message: string, threadId: string }`

The server passes `resourceId` and `threadId` to `agent.stream()`. Mastra fetches conversation history from the DB automatically.

```typescript
const stream = await chatAgent.stream(
  [{ role: 'user', content: message }],
  { resourceId: 'local-user', threadId, memoryOptions: { lastMessages: 20 } }
)
```

### GET /api/threads/:threadId

New endpoint. Returns stored messages for the thread so the UI can restore history on page load.

```typescript
// Uses mastra.memory.query({ threadId, resourceId: 'local-user' })
// Returns: { messages: Array<{ role, content }> }
```

## Frontend Changes

### `src/web/main.tsx`

- **On mount**: read `threadId` from `localStorage` (generate + store if absent), fetch `/api/threads/:threadId`, populate message state.
- **On submit**: send `{ message, threadId }` instead of the full messages array.
- **New Chat button**: delete `chat_thread_id` from `localStorage`, generate a new `threadId`, reset message state to the initial greeting.

## Data Flow

```
Browser                          Server                        storage.db
  │                                 │                               │
  │── GET /api/threads/:threadId ──▶│── mastra.memory.query ───────▶│
  │◀─ { messages: [...history] } ───│◀─ stored messages ────────────│
  │                                 │                               │
  │── POST /api/chat ──────────────▶│── agent.stream(msg, threadId)─▶│
  │   { message, threadId }         │   (Mastra reads+writes history)│
  │◀─ SSE deltas ───────────────────│                               │
```

## Out of Scope

- Multi-user support
- Conversation list / history browser
- Message deletion
- Deployment (LibSQL file storage is local-only)
